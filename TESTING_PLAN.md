
# План тестирования исправлений в Telegram-боте

## Цель:
Проверить корректность исправления, связанного с загрузкой переменных окружения для интеграции с GitHub, а также убедиться, что основной функционал бота не был нарушен.

## Тестовые сценарии:

### 1. Проверка интеграции с GitHub (Happy Path)

- **Шаги:**
    1.  Создать файл `.env` в директории `bot` со следующим содержанием:
        ```
        BOT_TOKEN=<ваш_токен_бота>
        ADMIN_USER_ID=<ваш_id_админа>
        GITHUB_TOKEN=<ваш_токен_github>
        REPO_NAME=<имя_вашего_репозитория_в_формате_owner/repo>
        ```
    2.  Запустить бота (`make run-bot`).
    3.  В чате с ботом (под аккаунтом админа) выполнить команду "Добавить товар".
    4.  Пройти все шаги добавления товара (название, описание, цена, количество, фото).
- **Ожидаемый результат:**
    - Товар успешно добавляется.
    - В консоли, где запущен бот, появляются логи:
        - `Local catalog.json updated successfully.`
        - `catalog.json updated successfully on GitHub in repository <имя_вашего_репозитория>.`
    - В репозитории на GitHub появляется новый коммит с сообщением "Update catalog.json via bot".
    - Файл `docs/catalog/catalog.json` в репозитории содержит новый добавленный товар.

### 2. Проверка интеграции с GitHub (Negative Path - без токена)

- **Шаги:**
    1.  В файле `.env` закомментировать или удалить строки `GITHUB_TOKEN` и `REPO_NAME`.
    2.  Перезапустить бота.
    3.  Выполнить операцию, изменяющую каталог (добавить, редактировать, удалить товар).
- **Ожидаемый результат:**
    - Операция (добавление, редактирование, удаление) успешно выполняется для локального каталога.
    - В консоли, где запущен бот, появляются логи:
        - `Local catalog.json updated successfully.`
        - `GitHub token or repository name not provided. Skipping GitHub update.`
    - Новый коммит в репозитории на GitHub **не появляется**.

### 3. Регрессионное тестирование основного функционала

- **Шаги:**
    1.  **Просмотр каталога:** Нажать кнопку "🛍 Каталог".
    2.  **Создание заказа:** Выбрать товар из каталога и нажать "Заказать этот товар", пройти все шаги.
    3.  **Обратная связь:** Нажать "💬 Написать менеджеру" и отправить сообщение.
    4.  **Редактирование товара:** Через "Список товаров" выбрать товар и отредактировать одно из его полей (например, цену).
    5.  **Удаление товара:** Через "Список товаров" удалить товар, подтвердив действие.
- **Ожидаемый результат:**
    - Все перечисленные функции работают так же, как и до внесения изменений. Бот не "падает", FSM-сценарии завершаются корректно.

## Итог:
Если все тесты пройдены успешно, исправление можно считать корректным и не нарушившим существующую функциональность.
