
# Feature Readiness Report: Project AM Muse Telegram Bot

## 1. Архитектурный обзор

- **Telegram Bot Library:** `aiogram` 3.x (судя по использованию `FSMContext`, `StatesGroup`, `Dispatcher` без `skip_updates`).
- **База данных:** Отсутствует. В качестве источника данных используется локальный JSON-файл `docs/catalog/catalog.json`. Это является и преимуществом (простота, отсутствие затрат), и недостатком (проблемы с конкурентным доступом, масштабируемостью).
- **Внешние API:**
  - **GitHub API:** Интеграция для обновления `catalog.json` в репозитории. Используется библиотека `PyGithub`. **Ключевое замечание:** в коде переменные `GITHUB_TOKEN` и `REPO_NAME` не загружаются из окружения, что делает эту функцию нерабочей "из коробки".
- **Асинхронность:** Используется `asyncio`, что стандартно для `aiogram`. Присутствует `asyncio.Lock()` для предотвращения гонок состояний при обработке заказов и обновлении каталога — это хороший защитный механизм.

## 3. Критические замечания (Blockers)

1.  **Нерабочая интеграция с GitHub:** Основная проблема, мешающая CI/CD и синхронизации данных. Бот работает только с локальной копией `catalog.json`, но сайт на GitHub Pages не увидит этих изменений. Переменные `GITHUB_TOKEN` и `REPO_NAME` должны быть загружены из `.env` файла, как и `BOT_TOKEN`.
2.  **Отсутствие персистентности FSM:** Состояния хранятся в памяти. При перезапуске бота все текущие операции пользователей (добавление товара, заказ) будут сброшены. Для продакшена необходимо использовать персистентное хранилище (например, `aiogram.fsm.storage.redis.RedisStorage`).
3.  **Ошибка в логике редактирования изображений:** Функция редактирования портит данные в `catalog.json`, заменяя массив путей к фото на одну текстовую строку. Это требует немедленного исправления.

## 4. Рекомендации по следующим шагам (Next Actions)

1.  **Настроить и активировать интеграцию с GitHub:**
    - Добавить загрузку `GITHUB_TOKEN` и `REPO_NAME` из переменных окружения (файл `.env`).
    - Протестировать полный цикл: добавление товара в боте -> автоматический коммит в репозиторий -> обновление сайта на GitHub Pages.
2.  **Исправить баг с редактированием изображений:**
    - Заменить текущую логику на запуск нового FSM-сценария, аналогичного добавлению изображений при создании товара.
3.  **Внедрить персистентное хранилище для FSM:**
    - Добавить в `requirements.txt` зависимость `redis`.
    - Настроить `RedisStorage` для `Dispatcher` для сохранения состояний между перезапусками.
4.  **Улучшить валидацию и обработку ошибок:**
    - Добавить валидацию на вводимые данные (например, что цена и количество > 0).
    - В сценарии добавления товара добавить обработку ситуации, когда одно из изображений не может быть загружено (например, отменить всю операцию или предложить повторить).
5.  **Добавить типизацию и форматирование:**
    - Провести форматирование кода с помощью `black` или `ruff format`.
    - Добавить type hints ко всем функциям для улучшения читаемости и поддержки кода.
6.  **Рефакторинг `update_catalog_data`:**
    - Разделить функцию на две: одна для локального сохранения, другая для выгрузки на GitHub. Это сделает код более чистым и тестируемым.
